## Istruzioni di load e store

Per architetture RISC queste sono le uniche istruzioni che danno accesso alla
memoria. Per CISC si può accedere anche in modo indiretto.

La memoria è solitamente indicizzata con un argomento che può trovarsi a seconda
dell'architettura:

- espresso come una costante (problematico su RISC per indirizzi 'grandi');
- indirizzo in un registro (base);
- indirizzo in un registro + un offset costante (spiazzamento);
- indirizzo in un registro + un offset in un altro registro (indice) (solo su
  CISC);

## ISA e ABI

L'ISA definisce le istruzioni riconosciute dalla CPU, il numero di registri ed
eventuali specializzazioni.

L'ABI è l'insieme delle convenzioni software che definiscono come usare i
registri.

Le convenzioni di chiamata definiscono, per una data CPU/architettura:

- come passare i parametri alle funzioni (registri o stack)
- quali registri preservare;
- quali contengono i valori di ritorno delle subroutine;

## Sintassi

Esistono principalmente 2 modi per scrivere assembly:

- sintassi Intel (usata nel corso): l'operando destinatario è il primo, i
  commenti sono `# commento`, etc...
- sintassi AT&T: l'operando destinatario è l'ultimo, i commenti sono
  `/* commento */`, etc...

## RISC-V

RISC-V è una moderna ISA RISC, sviluppata a Berkeley nel 2010 e resa open
source.

### Istruzioni aritmetiche

RISC-V prevede soltanto istruzioni aritmetiche a 3 operandi, ad esempio
`add a, b, c`. L'operando 'destinatario' è sempre il primo.

Operazioni più complesse tipo `a = b + c + d + e` si ottengono per combinazione:

```asm
add a, b, c
add a, a, d
add a, a, e
```

Gli operandi sono vincolati ad essere contenuti nei registri. Il numero di
registri è 32 da `x0` a `x31`, tutti a 64 bit.

Un gruppo di 64 bit è detto _double word_, uno di 32 bit è detto _word_.

**Esempio**: `f = (g + h) - (i + j)`

```asm
add x5, x20, x21   # x5 viene usato temporaneamente per salvare il risultato della somma g+h.
add x6, x22, x23   # Uguale per x6 con i+j.
sub x19, x5, x6    # x19 contiene il risultato finale.
```

#### Operandi immediati

Molte operazioni aritmetiche prevedono che almeno uno degli operandi sia
costante.

Ovviamente si può memorizzare la costante in un registro, ma ciò richiede
un'operazione in più.

RISC-V provvede istruzioni che permettono di operare con costanti, ad esempio
`f = f + 4` diventa `addi x22, x22, 4`. Queste istruzioni immediate sono
indicate dal suffisso `i`.

#### Registro 0

Per ottenere la costante 0 in un registro, di solito si usa un operazione di xor
tra i contenuti del registro: `xor x22, x22, x22` azzera il registro 22.

Quest'operazione è talmente comune che in RISC-V il registro `x0` è stato
dedicato a contenere sempre la costante 0.

### Istruzioni di memoria

RISC-V prevede solo 2 istruzioni per manipolare dati in memoria.

- `load` legge la memoria ad un indirizzo e la carica in un registro;
- `store` scrive il contenuto di un registro su un indirizzo in memoria;

Un indirizzo si specifica tramite un valore di base in un registro + uno
spiazzamento costante.

```asm
lw x9, 8 (x22)   # Carica una parola singola in x9 (dall'indirizzo in x22 + 8).
ld x9, 8 (x22)   # Carica una parola doppia.
```

**Esempio**: `a[12] = h + a[8]`

```asm
ld x9, 64 (x22)   # Legge il valore di `a` spostato di 8 elementi.
add x9, x21, x9   # Somma `h` (nel registro 21) al valore caricato.
sd x9, 96 (x22)   # Riscrive il valore in `a` + 12 elementi.
```

#### Register spilling

Il numero di registri di un processore è tipicamente minore del numero di
variabili usate in un programma, quindi bisogna per forza scriverle in memoria.
Il compilatore compie l'operazione di register spilling, cioè calcola quando
vengono usate le variabili ed esegue le operazioni di load e store al momento
più appropriato.

## Endianness

Per memorizzare una parola di 4 byte ci sono 2 modi:

- **little endian**: il byte più significativo va all'indirizzo più alto (usato
  in RISC-V, x86_64, etc...);
- **big endian**: il byte più significativo va all'indirizzo più basso (usato
  soprattutto su protocolli Internet);

## Rappresentazione delle istruzioni

Per rappresentare un'istruzione in codice macchina bisogna convertirla in un
numero.

Servono:

- un codice che indica univocamente quale operazione vogliamo eseguire;
- altri codici che indicano il numero del registro;
- eventuali argomenti numeri costanti;

In RISC-V ogni istruzione (con i suoi operandi) viene codificata in una word (32
bit):

| Lunghezza in bit | Scopo                       |
| ---------------- | --------------------------- |
| 7                | ID istruzione               |
| 5                | Secondo operando (registro) |
| 5                | Primo operando (registro)   |
| 3                | ID istruzione               |
| 5                | Risultato (registro)        |
| 7                | ID istruzione               |

Per le istruzioni immediate:

| Lunghezza in bit | Scopo                     |
| ---------------- | ------------------------- |
| 12               | Costante o indirizzo      |
| 5                | Primo operando (registro) |
| 3                | ID istruzione             |
| 5                | Risultato (registro)      |
| 7                | ID istruzione             |

La larghezza fissa delle istruzioni permette di creare un processore altamente
efficiente, ma con limiti sul numero di istruzioni, sul numero di registri e
sulle modalità di indirizzamento.
