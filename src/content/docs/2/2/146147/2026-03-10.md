## Pthreads

The Pthreads library is a POSIX standard API for thread creation and
synchronization. So it specifies the behavior of the library but implementation
is left to the developers.

```c
#include <pthread.h>
#include <stdio.h>
#include <stdlib.h>

// This variable is shared between the main and the worker thread.
int sum;

// Function that runs in the separate thread.
void* runner(void *param) {
    int i, upper = atoi(param);
    sum = 0;

    for (i = 0; i < upper; ++i) {
        sum += i;
    }

    // Could return a pointer with a return value, but we use the sum variable
    // for simplicity.
    pthread_exit(NULL);
}

int main(int argc, char* argv[]) {
    // Pointer to thread id;
    pthread_t tid;
    // Thread attributes;
    pthread_attr_t attr;

    // Initializes thread attributes with default values.
    pthread_attr_init(&attr);
    // Creates the thread and runs the runner function with argv[1] as param.
    pthread_create(&tid, &attr, runner, argv[1]);
    // Waits for the thread to call pthread_exit;
    pthread_join(tid, NULL);

    printf("sum = %d\n", sum);

    return 0;
}
```

## Implicit threading (OpenMP)

Explicit thread handling becomes difficult for non-trivial examples. Modern
approaches use the compiler to automatically insert code that spawns and waits
for threads to complete.

OpenMP uses the `#pragma omp parallel` compiler directive to automatically
parallelize a region of code:

```c
#include <omp.h>
#include <stdio.h>

int main() {
    printf("sequential code\n");

    // Creates as many threads as the system has available by default.
    // There are more directives to customize its behavior.
    #pragma omp parallel
    {
        printf("parallel code\n");
    }

    printf("sequential code\n");

    return 0;
}
```

:::tip

Compile with `gcc -fopenmp file.c`

:::

## CPU scheduling

Maximum CPU utilization is obtained with multiprogramming when:

- there are no context switches;
- there is no I/O waiting;

The usual process consists of a repeating cycle of CPU burst followed by an I/O
burst. A basic CPU scheduler selects from the processes in the ready queue and
allocates a CPU core to one of them.

CPU scheduling decisions take place when a process:

- switches from running to waiting state;
- switches from running to ready state;
- switches from waiting to ready state;
- terminates;

Under **non-preemptive** scheduling, the process keeps the CPU until it releases
it either by terminating or by switching to the waiting state. Modern OSes use
**preemptive** scheduling that allows the scheduler to interrupt a process and
switch to another at any point in time.

:::warn

Preemptive scheduling can result in race conditions, especially when data is
shared among threads or processes.

:::

The **dispatcher** is the module that controls what runs on the CPU. It handles
context switching and kernel to user mode switching. Dispatch latency is the
time it takes for the dispatcher to stop one process and start another.

### Scheduling algorithms

- First-come, first-served: executes processes in the order they arrive.
  Vulnerable to convoy effect: if first process is slow, the others wait a lot.
- Shortest job first: executes the job that takes less time first. It's
  difficult to know how long a process will take, but it can be approximated
  with an exponential average of its past data.
- Shortest remaining time first: this is the preemptive version of SJF. Whenever
  a process arrives in the ready queue, the decision is re-made using SJF.
- Round Robin: each process gets a small amount of CPU time ($q$), then it's
  preempted and added at the end of the ready queue. $q$ can't be too high
  (becomes FCFS) or too low (too much overhead).

#### Priority scheduling

A priority number is associated with each process. The CPU is allocated first to
the process with the highest priority.

There is a problem of starvation, where processes with low priority might never
run. The solution is aging: increase the priority as time passes.

For processes with the same priority, round-robin can be used.
